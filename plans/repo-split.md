# The Dungeon Notebook — План развития архитектуры

На данный момент проект — единый монорепозиторий. По мере роста планируется декомпозиция:

## Потенциальные репозитории

| Репозиторий | Состав | Причина выноса |
|-------------|--------|----------------|
| `tdn-auth` | `auth-service` | Заменяемость (Keycloak, LDAP). Свои RSA-ключи. |
| `tdn-uploads` | `uploads-service` | Уже отдельный Docker Compose. Свой lifecycle. |
| `tdn-admin` | `admin/` | Своя экосистема (Flask, Jinja2, CSS). Другая модель авторизации. Свой lifecycle. |

## Остаётся в `tdn-core`

| Состав | Причина |
|--------|---------|
| `api-gateway` | Точка входа, composite-хендлеры |
| `users-service` | Тесная связка с gateway и campaign-service |
| `campaign-service` | Бизнес-ядро |
| `monitoring/` | Инфраструктурные конфиги |

## Принципы декомпозиции

Сервис выносится в отдельный репозиторий, если выполняется хотя бы одно:

1. **Заменяемость** — сервис можно заменить на другой без изменения остальных
2. **Независимый lifecycle** — свой ритм релизов
3. **Независимые разработчики** — отдельный человек/команда
